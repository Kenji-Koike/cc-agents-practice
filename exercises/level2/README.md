# Level 2: 初めてのカスタムエージェント

## 目標

- エージェント定義ファイルの構造を理解する
- シンプルなカスタムエージェントを作成する
- 作成したエージェントを呼び出して動作を確認する

---

## Step 2-1: エージェント定義ファイルの構造

カスタムエージェントは以下の形式の Markdown ファイルで定義します。

```markdown
---
name: エージェント名
description: いつこのエージェントを使うかの説明（重要！）
tools: Read, Grep, Glob  # 使えるツールをカンマ区切りで列挙
---

ここにエージェントへのシステムプロンプトを書く。
このエージェントが何をする存在なのか、どう振る舞うかを記述する。
```

**`description` が重要な理由:**
Claude Code は `description` を見て、どのエージェントを自動的に選択するか判断します。
「〜する時に使う」という形式で書くのが効果的です。

---

## Step 2-2: ファイル一覧エージェントを作成する

以下のエージェントを作成してみましょう。

```
.claude/agents/file-lister.md を以下の内容で作成してください：

---
name: file-lister
description: ファイルやディレクトリの構造を調査・一覧表示する時に使う
tools: Glob, Read, Bash
---

あなたはファイルシステムの調査専門エージェントです。
ユーザーが指定したパスやパターンに基づいてファイルを探し、
わかりやすく整理して報告します。

報告形式：
- ファイル数の合計を最初に示す
- ディレクトリ構造をツリー形式で表示する
- 各ファイルの行数も可能な範囲で報告する
```

**実際にClaude Codeで実行するプロンプト例:**
```
.claude/agents/file-lister.md を上記の内容で作成してください
```

---

## Step 2-3: 作成したエージェントを確認する

```
/agents
```

`file-lister` が一覧に表示されるか確認しましょう。

---

## Step 2-4: エージェントを呼び出す

```
file-lister エージェントを使って、このリポジトリの exercises ディレクトリを調査してください
```

**確認ポイント:**
- `file-lister` エージェントが呼び出されましたか？
- 定義したシステムプロンプト通りの形式で報告されましたか？
- 使えるツールが `Glob, Read, Bash` に制限されていましたか？

---

## Step 2-5: description の効果を確認する

description を使った自動選択を試してみましょう。

```
exercises フォルダ内のファイルを全部見せてください
```

エージェントを明示的に指定せずに実行しました。
`file-lister` が自動的に選ばれましたか？

---

## チャレンジ課題

`greeting-agent.md` を作成してください。要件:
- name: `greeting-agent`
- tools: なし（空欄でOK）
- システムプロンプト: ユーザーの名前を聞いて、その人の好きなプログラミング言語に合わせた挨拶をする

ヒント: tools を空にするとどうなるか確認してみましょう。

---

## まとめ

- エージェントは frontmatter (---) + システムプロンプトで定義する
- `description` は自動選択の判断基準になる
- `tools` でエージェントができることを制限できる
