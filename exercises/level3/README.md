# Level 3: ツール制限とスコープ

## 目標

- ツール制限によるセキュリティとフォーカスを理解する
- 読み取り専用エージェントを作成する
- エージェントのスコープ（独立した実行コンテキスト）を体感する

---

## Step 3-1: なぜツールを制限するのか？

エージェントのツールを制限する理由は2つあります。

**1. セキュリティ:** 分析エージェントにファイル書き込みを禁止する
**2. フォーカス:** 専門タスクに集中させ、余計なことをさせない

例：
- コードレビューエージェント → `Read, Grep, Glob` のみ（書き込み不可）
- デプロイエージェント → `Bash` のみ（ファイル読み書き不可）
- ドキュメント生成エージェント → `Read, Grep, Write` のみ

## Step 3-2: 練習用コードを用意する

まず練習に使うコードファイルを作成しましょう。

```
exercises/level3/sample/ ディレクトリに以下の3ファイルを作成してください：

1. calculator.py
   - add, subtract, multiply, divide 関数を実装
   - divide はゼロ除算の場合 ValueError を raise する
   - 各関数には型ヒントを付ける（戻り値のヒントは省略）

2. user_manager.py
   - User クラス（name, email, age フィールド）
   - UserManager クラス（add_user, get_user, list_users メソッド）
   - age のバリデーション（0〜150）なし（バグとして意図的に省略）

3. utils.py
   - format_date(date_str: str) 関数：YYYY-MM-DD を MM/DD/YYYY に変換
   - is_valid_email(email: str) 関数：簡単な正規表現でメール検証
   - truncate_string(s: str, max_len: int) 関数：文字列を切り詰める（日本語対応なし）
```

## Step 3-3: 読み取り専用の静的解析エージェントを作成する

```
.claude/agents/static-analyzer.md を以下の内容で作成してください：

---
name: static-analyzer
description: Pythonコードの静的解析、バグ発見、改善提案をする時に使う
tools: Read, Grep, Glob
---

あなたはPythonコードの静的解析専門家です。
コードを**読むだけで**問題点を発見し、改善提案をします。
ファイルの変更は一切行いません。

## 解析観点
1. バグの可能性（境界値、型エラー、例外処理漏れ）
2. セキュリティリスク（インジェクション、バリデーション不足）
3. パフォーマンス問題
4. Pythonのベストプラクティス違反

## 出力形式
各問題について：
- **ファイル名:行番号** — 問題の概要
- 問題の詳細説明
- 修正提案（コードスニペット付き）

重大度を 🔴高 / 🟡中 / 🟢低 で示すこと。
```

## Step 3-4: 静的解析エージェントを実行する

```
static-analyzer エージェントを使って exercises/level3/sample/ を解析してください
```

**確認ポイント:**
- エージェントはファイルを変更しようとしましたか？（しないはずです）
- `Write` や `Edit` ツールが使われましたか？（使われないはずです）
- user_manager.py の age バリデーション漏れを発見しましたか？

---

## Step 3-5: スコープの独立性を確認する

エージェントはメインの会話コンテキストから独立して動きます。

```
# まずメインで変数を定義するような会話をする
私の名前は田中です。これを覚えておいてください。

# 次にエージェントを呼び出す
static-analyzer エージェントを使って calculator.py を解析し、
解析の最後に「私の名前は何ですか？」と答えてください
```

**確認ポイント:**
- エージェントは「田中」を知っていましたか？
- メインコンテキストの情報がエージェントに漏れない理由は何ですか？

## Step 3-6: Write ツールを持つエージェントとの比較

ツールがあると何が変わるか確認します。

```
.claude/agents/code-fixer.md を作成：

---
name: code-fixer
description: コードのバグを実際に修正する時に使う
tools: Read, Grep, Glob, Edit, Write
---

あなたはコードの修正専門家です。
バグを発見したら実際にファイルを編集して修正します。
修正後は変更内容のサマリーを報告してください。
```

作成後:
```
code-fixer エージェントを使って user_manager.py の age バリデーションを追加してください
```

## チャレンジ課題

以下の要件でエージェントを作成してください:
- **name:** `dependency-auditor`
- **目的:** `requirements.txt` や `pyproject.toml` を読んで依存関係の問題を報告する
- **制約:** ファイル読み取りのみ（インストールや変更は不可）
- **出力:** パッケージ名・バージョン・既知リスクのテーブル形式

---

## まとめ

- `tools` でエージェントの行動範囲を明示的に制限できる
- 読み取り専用エージェントはコードを変更できないため安全
- エージェントはメインコンテキストから独立した実行環境を持つ
