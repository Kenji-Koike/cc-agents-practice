# Level 1: エージェントの基礎理解

## 目標

- `/agents` コマンドの基本的な使い方を理解する
- ビルトインエージェントの種類と役割を把握する
- エージェントがどのような仕組みで動作するかを理解する

---

## Step 1-1: `/agents` コマンドを実行してみる

Claude Code のプロンプトで以下を実行してください。

```
/agents
```

**確認ポイント:**
- どのようなエージェントが表示されますか？
- 各エージェントに何が表示されていますか？

---

## Step 1-2: ビルトインエージェントを確認する

Claude Code にはいくつかのビルトインエージェントがあります。
以下のプロンプトを試してみましょう。

```
利用可能なエージェントの種類と、それぞれの用途を教えてください
```

**確認ポイント:**
- `general-purpose` エージェントは何に使われますか？
- `Explore` エージェントの特徴は何ですか？

---

## Step 1-3: エージェントが使うツールを観察する

以下のプロンプトを試して、エージェントがどのツールを使うか観察しましょう。

```
このリポジトリの構造を調査してください（Exploreエージェントを使って）
```

**確認ポイント:**
- どのツールが呼び出されましたか？
- メインの会話と何が違いますか？

---

## Step 1-4: エージェント定義ファイルの場所

カスタムエージェントは `.claude/agents/` に置きます。
現在の状態を確認しましょう。

```bash
# ターミナルで実行
ls -la .claude/agents/ 2>/dev/null || echo "まだ .claude/agents/ ディレクトリは存在しません"
```

---

## まとめ

- `/agents` コマンドで利用可能なエージェントを確認できる
- ビルトインエージェントはそれぞれ異なるツールセットと用途を持つ
- カスタムエージェントは `.claude/agents/` に Markdown ファイルで定義する
